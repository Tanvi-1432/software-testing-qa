# CS-320 Portfolio Reflection

## How can I ensure that my code, program, or software is functional and secure?

I make sure my code is functional by testing it against the project requirements and creating unit tests for both valid and invalid scenarios. Throughout this course, I used JUnit to verify that each class behaved as expected and handled invalid input correctly. I also used exception testing to make sure incorrect data was rejected instead of causing unexpected behavior. From a security standpoint, validating user input is one of the most important steps because it prevents invalid or unexpected values from being accepted by the application.

## How do I interpret user needs and incorporate them into a program?

I start by carefully reviewing the requirements and breaking them into smaller, testable pieces. Before writing any code, I identify what the application needs to do and create test cases that verify each requirement. During this project, every validation rule, update method, and service function was implemented directly from the customer requirements. This approach helped ensure that the final application matched what the customer requested instead of making assumptions during development.

## How do I approach designing software?

I prefer to keep the design simple and build one component at a time. I focus on writing clean, maintainable code that solves the required problem without adding unnecessary complexity. I also like to test each feature as I develop it instead of waiting until the end of the project. This approach makes it easier to catch bugs early, verify that new changes do not break existing functionality, and produce more reliable software.
